# ✦ AI-школа

Красивый интерактивный учебный сайт по нейросетям. Проект полностью статический и готов к GitHub Pages.

## Что уже работает

- 11 образовательных программ;
- 465 уроков и 2 161 промпт;
- поиск по курсам, модулям и материалам;
- отметки пройденных уроков и общий прогресс;
- библиотека промптов с поиском, категориями и копированием;
- тёмная и светлая темы;
- адаптация под телефон, планшет и компьютер;
- анимированный фон, карточки, счётчики, плавные появления и конфетти;
- офлайн-кэш после первого открытия;
- все исходные PDF, Markdown, CSV и Excel-файлы внутри папки `materials`;
- TypeScript-исходник в `src/app.ts` и готовый JavaScript в `assets/app.js`.

## Публикация на GitHub Pages

### Способ 1 — через готовый GitHub Actions

1. Создайте новый репозиторий на GitHub.
2. Загрузите **всё содержимое этой папки** в корень репозитория.
3. Откройте `Settings → Pages`.
4. В поле `Source` выберите **GitHub Actions**.
5. Откройте вкладку `Actions` и дождитесь завершения процесса `Deploy AI School to GitHub Pages`.
6. Готовая ссылка появится в `Settings → Pages`.

Файл автоматической публикации уже находится здесь: `.github/workflows/pages.yml`.

### Способ 2 — публикация из ветки

1. Загрузите файлы в ветку `main`.
2. Откройте `Settings → Pages`.
3. Выберите `Deploy from a branch`.
4. Выберите ветку `main` и папку `/(root)`.
5. Нажмите `Save`.

## Как проверить на компьютере

Можно открыть `index.html` двойным щелчком. Для полноценной проверки офлайн-кэша запустите локальный сервер:

```bash
python -m http.server 8080
```

Затем откройте `http://localhost:8080`.

На Windows также можно запустить файл `ЗАПУСТИТЬ_ЛОКАЛЬНО.bat`.

## Изменение дизайна

Основные цвета находятся в начале файла `assets/styles.css`:

```css
:root {
  --brand: #8b5cf6;
  --brand-2: #22d3ee;
}
```

После изменения TypeScript выполните:

```bash
npm install
npm run build
```

Для простой публикации ничего собирать не нужно: готовый `assets/app.js` уже добавлен.

## Важное замечание

Прогресс обучения сохраняется в браузере пользователя через `localStorage`. Он не передаётся на сервер и не синхронизируется между разными устройствами.
